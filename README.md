# myQRID Web MVP

A simple production-deployable myQRID MVP using the current stack only:

- Node.js
- Express
- Firebase Firestore
- HTML/CSS/JS
- npm
- Render

No Next.js, React, TypeScript, MongoDB, Docker or microservice runtime is required for this MVP.

## What works end to end

1. Landing page opens on `/`.
2. User creates a profile with name, phone, WhatsApp, email, bio and profile image URL.
3. Backend validates the payload and saves the profile to Firebase Firestore when Firebase env vars are configured.
4. Backend auto-generates:
   - `username`
   - public profile URL: `/u/:username`
   - tag URL: `/t/:slug`
   - profile QR URL
   - tag QR URL
5. Public profile page works at `/u/:username`.
6. Tag scan page works at `/t/:slug`.
7. Public pages include:
   - Call
   - WhatsApp
   - Save Contact / VCF download
   - Copy profile link
8. UI is mobile responsive.
9. Loading and error states are implemented in the browser.
10. Firebase status is visible through `/api/status` and `/health`.
11. If Firebase is not configured, the app uses temporary in-memory fallback storage so local UI testing still works. Use Firebase env vars on Render for production persistence.
12. Admin page works at `/admin` with password protection.
13. Admin can view all profiles and delete a profile.
14. QR PNGs are generated by the Express backend at `/api/qr?data=...`.
15. `npm start` launches the production MVP.

## Local setup

### 1. Install dependencies

```bash
npm install
```

### 2. Optional: create local environment file

Create `.env` or set environment variables in your shell. Firebase is optional for local UI testing but required for production persistence.

```txt
FIREBASE_PROJECT_ID=your-firebase-project-id
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxxx@your-project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY=-----BEGIN PRIVATE KEY-----\nYOUR_KEY_HERE\n-----END PRIVATE KEY-----\n
PUBLIC_BASE_URL=http://localhost:3000
ADMIN_PASSWORD=change-this-admin-password
SETUP_SECRET=optional-legacy-setup-secret
PORT=3000
```

### 3. Start locally

```bash
npm start
```

Open:

```txt
http://localhost:3000/
```

## Firebase setup

1. Open Firebase Console.
2. Create or select your Firebase project.
3. Enable Firestore Database.
4. Go to Project settings → Service accounts.
5. Generate a new private key.
6. Copy these values into Render environment variables:
   - `FIREBASE_PROJECT_ID`
   - `FIREBASE_CLIENT_EMAIL`
   - `FIREBASE_PRIVATE_KEY`
7. Store the private key as one environment variable with escaped newlines:

```txt
-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n
```

Firestore collections are created automatically when the first profile is saved:

- `profiles`
- `tags`
- `scan_logs`

## Render deployment

1. Push this repository to GitHub.
2. Create a Render Web Service.
3. Connect the GitHub repository.
4. Use these settings:

```txt
Runtime: Node
Build command: npm install
Start command: npm start
```

5. Add environment variables:

```txt
FIREBASE_PROJECT_ID=your-firebase-project-id
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxxx@your-project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY=-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n
PUBLIC_BASE_URL=https://your-render-service.onrender.com
ADMIN_PASSWORD=make-a-long-private-password
SETUP_SECRET=make-a-long-private-password
```

6. Deploy.
7. Open the Render root URL:

```txt
https://your-render-service.onrender.com/
```

## API routes

### Status

```txt
GET /health
GET /api/status
```

### Profiles

```txt
POST /api/profiles
GET  /api/profiles/:username
```

Create profile JSON:

```json
{
  "name": "Preet Mahant",
  "phone": "+919876543210",
  "whatsapp": "+919876543210",
  "email": "hello@example.com",
  "bio": "Founder of myQRID",
  "image_url": "https://example.com/photo.jpg"
}
```

### Tags

```txt
GET /api/tags/:slug
```

### QR generation

```txt
GET /api/qr?data=https%3A%2F%2Fexample.com%2Fu%2Fpreet
```

### Admin

```txt
GET    /admin
GET    /api/admin/profiles
DELETE /api/admin/profiles/:username
```

Admin API requests must include:

```txt
x-admin-password: YOUR_ADMIN_PASSWORD
```

## Manual verification checklist

After `npm start`:

1. Open `/` and create a profile.
2. Confirm the result card shows username, profile URL, tag URL and two QR codes.
3. Open `/u/:username` and test Call, WhatsApp, Save Contact and Copy profile link.
4. Open `/t/:slug` and confirm it shows the same owner profile.
5. Open `/admin`, enter `ADMIN_PASSWORD`, view profiles and delete a test profile.
6. Open `/api/status` and confirm `storage` is `firebase` on Render after Firebase env vars are configured.

## Notes about Firebase fallback

When Firebase credentials are missing or invalid, `/api/status` returns `storage: "memory_fallback"`. This is only for local/demo testing. Data will disappear when the Node process restarts. For production on Render, configure Firebase so `/api/status` returns `storage: "firebase"`.
